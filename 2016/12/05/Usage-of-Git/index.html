git 分支管理

git 分支管理基本原则, master分支仅用于实际开发环境的分支, 用于版本的管理和最后的完成版本开发的话, 使用分支develop, 因此保持两个常用分支, master和develop. 若需要调试bug等操作, 设置一个临时的分支, 用完即删除掉即可.
